Definições
API é o acrônimo de Application Programming Interface ou, em português, Interface de
Programação de Aplicativos.
Informações Gerais
A API está localizada no endereço https://demo.aplis.inf.br/api/integracao.php
A troca das informações se dá através do envio e recebimento de conteúdos JSON.
O conteúdo enviado à API, assim como o retorno, possuirá a seguinte estrutura:
{
ver: numero_da_versao,
cmd: nome_do_comando,
dat: dados (ver comandos suportados)
}

Comandos Disponíveis
A seguir, os comandos suportados pela API.
● Os campos com asterisco “*” são de preenchimento obrigatório.
● Campos data são expressos no formato “AAAA-MM-DD”.
● Campos hora são expressos no formato “HH:MM:SS”, no padrão 24H.
● Campos data/hora são expressos no formato “AAAA-MM-DD HH:MM:SS”
● Campos numéricos não utilizar separador de milhar e utilizar ponto “.” para separação de
decimais.
1 – Admissão (comando admissaoSalvar)
Efetua a gravação (inclusão ou alteração) de uma requisição, como na tela da admissão do apLIS.
Dados:
Campo Tipo Descrição
codRequisicao Texto Código de identificação da requisição.
Se vazio, uma requisição será criada e um código será gerado.
Se preenchido, uma requisição será criada com o código fornecido.
Caso já exista o código, a requisição existente será atualizada.
idLaboratorio* Numérico Código de identificação do laboratório
idUnidade* Numérico Código de identificação da unidade de recebimento do material
idUnidadeExt Texto Código de identificação externo da unidade (conforme de/para)
idPaciente* Numérico Código de identificação do paciente
idPacienteExt Texto Código de identificação externo do paciente (no cliente)
dtaColeta Data Data de coleta do material
idConvenio* Numérico Código de identificação do convênio
idConvenioExt Texto Código de identificação externo do convênio (conforme de/para)
idPlano Numérico Código de identificação do plano/modalidade do convênio do
paciente
idPlanoExt Texto Código de identificação externo do plano/modalidade do convênio
(conforme de/para)
numGuia Texto, 50 Número da guia do convênio
idLocalOrigem* Numérico Código de identificação do local de origem
idLocalOrigemExt Texto Código de identificação externo do local de origem (conforme
de/para)
idLocalSetor Numérico Código de identificação do setor do local de origem
numExterno Texto, 512 Número de identificação externo do exame, no local de origem
idFontePagadora* Numérico Código de identificação da fonte pagadora
idFontePagadoraExt Texto Código de identificação externo da fonte pagadora(conforme
de/para)
idMedico* Numérico Código de identificação do médico solicitante
idCBO Numérico Código CBO do médico solicitante
idMedicoPart Numérico Código de identificação do médico participante
idExame* Numérico Código de identificação do exame principal solicitado
idExameExt Texto Código de identificação externo do exame principal solicitado
(conforme de/para)
examesConvenio* Array
Numérico
Lista de códigos de identificação dos exames a serem cobrados da
fonte pagadora
testesPatClinica Array
Numérico
Lista dos códigos dos testes de patologia clínica que serão realizados
dadosClinicos Texto, 3000 Dados clínicos do exame

2 – Cadastro de Pacientes (comando pacienteSalvar)
Efetua a gravação (inclusão ou alteração) dos dados de um paciente.
Dados:
Campo Tipo Descrição
idPaciente Numérico Código de identificação do paciente
idPacienteExt Texto Código de identificação externo do paciente (no cliente)
nome* Texto, 255 Nome do paciente
dtaNasc Data Data de nascimento
sexo Texto, 1 F ou M
nomeMae Texto, 255 Nome da mãe
rg Texto, 15 RG do paciente
rgOrgao Texto, 10 Órgão emissor do RG
rgUF Texto, 2 Sigla do estado onde o RG foi emitido
cpf Texto, 15 CPF do paciente
cpfAusente Numérico. 1 Opções: 1 - Esqueceu documento 2 - Estrangeiro 3 - Não possui
idConvenio Numérico Código de identificação do convênio
matConvenio Texto, 30 Código de matrícula/nº carteirinha do paciente no convênio
validade Data Validade da carteirinha
cep Numérico, 8 CEP do paciente
logradouro Texto, 255 Logradouro (Rua, Av, etc) do endereço
numEndereco Texto, 20 Número do logradouro
comEndereco Texto, 255 Complemento do endereço
bairro Texto, 255 Bairro do endereço
cidade Texto, 255 Cidade do endereço
uf Texto, 2 UF da cidade
telFixo Texto, 20 Telefone fixo
telCelular Texto, 20 Telefone celular
email Texto, 255 E-mail
Retorno:
Campo Tipo Descrição
sucesso Numérico, 1 Retornará 1 em caso de sucesso e os demais campos virão
preenchidos.
Em caso de erro, retornará 0 e os campos codErro e msgErro.
idPaciente Numérico Código de identificação do paciente
